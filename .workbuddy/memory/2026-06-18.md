## Bug 修复：交易流水返回了所有用户的数据

### 问题
`GET /api/transactions/my` 返回了全部 17 条记录（包含 admin/player2/player3/qauser99 的），导致 player1 看到多个"注册奖励"(100/200/300/500/1000)

### 根因
`transactions.controller.ts` 中获取 userId 的方式：
```typescript
const userId = (req as any).user.id; // ❌ JWT payload 字段是 userId，不是 id
```
JWT payload 定义 `JwtPayload { userId: number; role: Role }`，所以 `req.user.id` 返回 `undefined`。Prisma 遇到 `where: { userId: undefined }` 会忽略该过滤条件，返回全部数据。

### 修复
```typescript
const userId = (req as AuthRequest).user!.userId; // ✅ 正确字段
```
