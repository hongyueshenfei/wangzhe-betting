# 王者荣耀竞猜 · AI 辅助编码方法论

> 2026-06-22 建立。基于"约束 + 认知 + 验证"三层体系，让 AI 产出可预期、可验证、可持续。

---

## 核心理念

**AI 编码的效率不取决于模型有多聪明，而取决于你的工程基础设施有多扎实。**

具体到这个项目：

```
AI 编码效率 ≈ 约束清晰度 × 认知完整度 × 验证完备度
```

---

## 三层体系

### 第一层：约束（Rule）— 让 AI 知道"不能做什么"

**文件**：`.cursorrules`（20 条规则）

AI 在写代码时会自动读取，不需要人提醒。覆盖 7 个维度：

| 维度 | 规则示例 | 效果 |
|------|---------|------|
| 架构分层 | Controller 禁止写业务逻辑 | AI 不再犯低级架构错误 |
| 错误处理 | 所有业务错误用 AppError | 统一错误风格，前端可预期 |
| API 规范 | 必须用 response.ts 的工具函数 | 响应格式统一，前端直接消费 |
| 安全 | JWT 在 middleware 统一校验 | 不会遗漏认证守卫 |
| 命名 | 文件命名 xx.service.ts | 新文件按约定创建 |
| 代码质量 | 函数不超过 60 行 | AI 产出可读性有底线 |
| AI 作用域 | 未经确认不改非目标文件 | 防止"顺手"改坏代码 |

### 第二层：认知（AGENTS.md）— 让 AI 知道"项目是什么"

**文件**：`AGENTS.md`

一张索引卡包含：技术栈、目录结构、架构分层、7 个数据模型、业务常量、12 个 API 模块、4 个工作流程、文件索引。

AI 遇到不确定时，按以下优先级查找：
1. `.cursorrules` — 硬约束
2. `docs/system_design.md` — 完整设计文档（1072 行）
3. 同类文件（如已有其他 Controller/Service 的实现模式）
4. 问人

### 第三层：验证（Tests）— 让 AI 的产出可验证

**31 个单元测试**，覆盖 4 个核心 Service：

| 模块 | 测试数 | 验证内容 |
|------|:------:|---------|
| settlement.service | 7 | 胜方赔付、负方无赔付、幂等性、弃赛退款、冠军投注比例分摊 |
| bets.service | 11 | 正常投注、余额不足、重复投注、比赛状态、封禁用户、最小值、赔率更新 |
| odds.service | 8 | 默认赔率、偏态赔率、范围约束、BASE_BET 保护、小数精度、管理员调整 |
| checkin.service | 5 | 签到奖励、重复签到、封禁用户、不存在用户、跨日签到 |

**测试纪律**：集成测试禁止 mock 数据库，使用独立 SQLite 测试库（`prisma/test.db`）。

## 5 步编码工作流（SOP）

这是每次用 AI 写代码的推荐流程：

```
Step 1: 写需求描述
  → 你写：什么功能、输入输出、边界条件
  → 不需要格式，说清楚就行

Step 2: AI 实现 + 写单测
  → AI 生成功能代码 + 对应的单元测试
  → 测试覆盖：Happy Path + Error Path + 边界条件

Step 3: 跑 lint
  → npx tsc --noEmit
  → 类型错误在早期修复

Step 4: 跑测试
  → npm test（31 个单测 + 后续新增）
  → 全绿再往下走

Step 5: 人工确认
  → 不逐行读代码，看：
    ① diff 关键路径（核心逻辑）
    ② 测试设计是否合理（有没有 mock 了不该 mock 的）
    ③ 架构分层是否正确
```

## CI 门禁

**配置**：`.github/workflows/ci.yml`

每次 push/PR 到 `main` / `master` 分支自动执行：
1. `npm ci` — 安装依赖
2. `npx prisma generate` — 生成 Prisma Client
3. `npx tsc --noEmit` — TypeScript 编译检查
4. `npx vitest run` — 全量单测

三级门禁，任何一级失败阻塞合并。

## 翻车预防清单

来自实战教训，写在 `.cursorrules` 中：

| 翻车场景 | 预防手段 |
|---------|---------|
| AI 改了不该改的文件 | "未经确认不得修改非目标目录" |
| 集成测试 mock 了数据库导致假绿 | "集成测试禁止 mock 数据库层" |
| 重构中 AI 混用新旧 API | "重构任务禁止 import 旧包路径" |
| 金额扣减并发问题 | 强制使用 `prisma.coins.decrement` |
